📅 스마트 학사일정 대시보드 (Smart Academic Schedule Dashboard)
엑셀 파일 하나로 학사 일정을 자동화하고 실시간 알람을 제공하는 실무형 통합 관리 대시보드입니다. 비개발자가 AI(Claude, Gemini)와의 협업을 통해 단일 HTML 파일로 구축한 성공적인 Vibe Coding 프로젝트입니다.

✨ 주요 기능 (Core Features)
1. 지능형 엑셀 파싱 & 자동화 엔진
스마트 파서: '시작일'과 '종료일'이 있는 리스트형 시트와, '구분'과 '기수'가 교차하는 매트릭스형 시트를 모두 자동으로 인식합니다.
업로드 투명성 리포트: 엑셀 업로드 시 총 행수, 성공 건수, 빈 행이나 오류로 제외된 건수를 팝업으로 상세히 보고합니다.
D-Day 자동 계산: '평가(시험)' 일정이 인식되면 시험일 기준 **D-14(공지 올리기)**와 **D-2(시험지 생성)** 업무가 달력에 자동으로 추가됩니다.
미응시자 독려 자동 생성: 중간고사·기말고사 일정에 한해 시험일 **D+4(미응시자 독려 안내)** 업무가 자동으로 달력에 추가됩니다.

2. 고급 시각화 및 UI/UX
Stitch Premium 디자인: 둥근 모서리와 다중 그림자 효과를 적용한 벤토 그리드(Bento Grid) 레이아웃을 채택했습니다.
이번 주 진행 업무 우선 배치: 예정된 평가, 공지 업무, 시험지 생성, 미응시자 독려 현황을 화면 최상단에서 즉시 확인할 수 있습니다.
13색 고유 컬러 시스템: 13개의 학사 카테고리(개강, 종강, 중간, 기말, 과제, 토론, 미응시자독려 등)에 고유한 색상을 부여하여 상단 필터와 달력 배지를 동기화했습니다.
기간 막대(Gantt-Bar) 표시: 시작일과 종료일이 다른 일정은 달력을 가로지르는 연속된 가로 막대 형태로 표시됩니다.
무한 스크롤: 데이터가 많아져도 잘리지 않도록 100vh 제한을 해제하고 시원한 스크롤을 제공합니다.

3. 정밀한 일정 관리 (CRUD) 및 이원화 메모
일정 통합 관리: 날짜를 클릭하면 해당 일의 일정 목록이 뜨며, 개별 일정마다 완료(취소선), 수정, 삭제 처리가 가능합니다.
타입별 독립 완료 처리: 하나의 시험 일정에서 파생된 공지(D-14), 시험지 생성(D-2), 미응시자 독려(D+4), 시험일 배지가 각각 독립적으로 완료 처리됩니다. 공지를 완료해도 시험일 배지는 영향을 받지 않습니다.
일일 메모 (노란 딱지): 특정 날짜의 빈 공간을 클릭해 메모를 저장하면 달력 숫자 옆에 '메모있음' 라벨이 표시됩니다.
특이사항 (빨간 점): 특정 일정 배지를 클릭해 특이사항을 저장하면 해당 배지 왼쪽 상단에 빨간 점이 표시됩니다 (기말고사에 적은 메모가 공지 일정에 전염되지 않도록 분리됨).

4. 스마트 알람 엔진
2중 알람 체계: 브라우저 푸시 알림과 화면 중앙 강제 팝업(Alert)을 동시에 사용하여 로컬 환경에서도 알람을 놓치지 않게 합니다.
캐치업 & 정시 알람: 대시보드 접속 시 '오늘 할 일'과 '미처리 업무'를 분리해 보고하며, 사용자가 설정한 시간에 10초 단위 정밀 타이머로 알람을 보냅니다.

5. 구글 앱스 스크립트 이메일 알림
매일 자동 발송: 구글 앱스 스크립트(Google Apps Script)의 시간 기반 트리거를 활용하여 매일 오후 1시에 오늘의 업무 현황을 이메일로 자동 발송합니다.
즉시 발송 버튼: 대시보드 상단의 [📧 이메일 발송] 버튼을 누르면 즉시 이메일이 발송됩니다 (업무가 없어도 발송).
브라우저 종료 무관: 앱스 스크립트 서버가 자동 실행하므로 대시보드 창이 닫혀있어도 이메일이 정상 발송됩니다.
설정 방법: 리포지토리의 학사일정_이메일알림_AppScript.js 파일 내 CONFIG 항목에 수신자 이메일을 설정한 뒤, Google Apps Script에 배포하여 웹 앱 URL을 대시보드 설정창에 입력합니다.

6. 반복 일정 등록
기간 + 요일 지정 등록: 시작일~종료일 범위와 반복 요일(일~토)을 선택하면 해당 기간 내 모든 해당 요일에 일정이 자동 생성됩니다. (예: 5/1~8/31 매주 화요일 → SMS 보내기 일정 자동 생성)
단일/반복 탭 전환: 기존 단일 등록과 반복 등록을 탭으로 분리하여 편리하게 전환할 수 있습니다.
요일 토글 UI: 요일 버튼을 클릭하면 파란색으로 강조되어 선택 상태를 직관적으로 확인할 수 있습니다.

7. 클라우드 연동 및 인수인계
Supabase 실시간 동기화: URL과 API 키를 설정하면 로컬 스토리지를 넘어 클라우드 DB와 연동되어 팀원 간 실시간 일정 공유가 가능합니다.
전체 초기화 버튼: 타 부서 배포 시, 브라우저에 남은 기존 사용자의 설정값과 API 키 등을 한 번에 파쇄하는 공장 초기화 기능을 탑재했습니다.

🛠️ 기술 스택 (Tech Stack)
Frontend: HTML5, CSS3, Vanilla JavaScript (단일 파일 구조)
Library: SheetJS (엑셀 파일 .xlsx, .csv 파싱)
Email: Google Apps Script (시간 기반 트리거 + 웹 앱 POST)
Backend / DB (Optional): Supabase (PostgreSQL 기반 실시간 데이터베이스)

🚀 설치 및 사용 방법 (How to Use)
본 리포지토리의 INDEX.html 파일을 다운로드하여 Chrome 등의 브라우저에서 실행합니다.
우측의 [엑셀 파일 업로드] 버튼을 눌러 학사일정 양식에 맞는 엑셀 파일을 업로드합니다.
(이메일 알림 설정) 학사일정_이메일알림_AppScript.js 파일을 script.google.com에 붙여넣고, CONFIG의 수신자 이메일을 수정한 뒤 웹 앱으로 배포합니다. createTrigger() 함수를 1회 실행하면 매일 오후 1시 자동 발송이 활성화됩니다. 배포 URL을 대시보드 [⚙️ 설정] 창의 Apps Script URL 항목에 입력합니다.
(팀 공유가 필요한 경우) [⚙️ 설정] 창을 열어 본인 팀의 Supabase URL과 API Key를 입력하여 실시간 동기화를 활성화합니다.
다른 팀에 템플릿으로 배포할 때는 설정창 하단의 [⚠️ 시스템 전체 초기화] 버튼을 눌러 개인 데이터를 지운 후 전달합니다.

📧 이메일 알림 상세 설정 (Email Notification Setup)
1. https://script.google.com 접속 → 새 프로젝트 생성
2. 이 리포지토리의 학사일정_이메일알림_AppScript.js 내용을 전체 복사하여 붙여넣기
3. CONFIG 섹션에서 수신자 이메일 주소 수정
4. 저장 후 [배포] → [새 배포] → 유형: 웹 앱 → 액세스: 모든 사용자 → 배포
5. 생성된 웹 앱 URL 복사
6. Apps Script 편집기에서 createTrigger() 함수 선택 후 실행 (1회만, 매일 1PM 트리거 생성)
7. 대시보드 [⚙️ 설정] → Apps Script URL 항목에 URL 붙여넣기 → 저장

v2.9.27 (이메일 동기화 개선): 일정 등록/수정/삭제/완료처리 시마다 Apps Script에 자동 동기화. 기존 페이지 로드 시 1회 동기화 방식에서 변경 → 1시 자동 이메일에 항상 최신 데이터 반영.
